# نظام التجارة الإلكترونية

## وصف المشروع

يهدف نظام التجارة الإلكترونية إلى توفير منصة متكاملة لعرض المنتجات وإدارة عمليات الشراء بطريقة سهلة وآمنة، حيث يمكن للمستخدمين تصفح المنتجات المتاحة، وإضافة المنتجات إلى الطلبات، ومتابعة حالة الطلب حتى استلامه. كما يوفر النظام لوحة تحكم خاصة بالإدارة لإدارة المنتجات والمستخدمين والتحكم بصلاحيات الدخول إلى النظام.

---

# أنواع المستخدمين

## 1. العميل (Customer)

يستطيع العميل القيام بالمهام التالية:

* إنشاء حساب وتسجيل الدخول إلى النظام.
* استعراض المنتجات المتاحة.
* البحث عن المنتجات حسب الاسم أو الفئة.
* تقديم طلب شراء للمنتجات.
* متابعة حالة الطلب.
* عرض سجل الطلبات السابقة.
* التواصل مع الإدارة من خلال نموذج الاتصال.

### حالات الطلب

تمر الطلبات بالمراحل التالية:

1. **قيد الانتظار (Pending)**
   عند إنشاء الطلب.

2. **تمت الموافقة (Approved)**
   بعد مراجعة الطلب من قبل الإدارة.

3. **تم الشحن (Shipped)**
   عند تجهيز الطلب وإرساله.

4. **تم التسليم (Delivered)**
   عند استلام العميل للطلب.

5. **ملغي (Cancelled)**
   في حال إلغاء الطلب.

---

## 2. الإدارة (Admin)

توفر لوحة التحكم للمسؤول مجموعة من الصلاحيات، منها:

### إدارة المنتجات

* إضافة منتجات جديدة.
* تعديل بيانات المنتجات.
* حذف المنتجات.
* إدارة المخزون والكميات المتاحة.
* تصنيف المنتجات حسب الفئات.

### إدارة الطلبات

* عرض جميع الطلبات.
* الموافقة على الطلبات.
* تغيير حالة الطلب.
* متابعة عمليات التسليم.
* إلغاء الطلبات عند الحاجة.

### إدارة المستخدمين

* عرض المستخدمين المسجلين.
* إضافة مستخدمين جدد.
* تعديل بيانات المستخدمين.
* حذف المستخدمين.
* التحكم بصلاحيات الدخول.

### إدارة الرسائل

* عرض الرسائل الواردة من العملاء.
* متابعة استفسارات العملاء والرد عليها.

---

# نظام الصلاحيات

يعتمد النظام على نوع المستخدم (Role-Based Access Control):

## صلاحية العميل

* استعراض المنتجات.
* إنشاء الطلبات.
* متابعة الطلبات الخاصة به فقط.
* تعديل بياناته الشخصية.

## صلاحية المدير (Admin)

* الوصول إلى لوحة التحكم.
* إدارة المنتجات.
* إدارة المستخدمين.
* إدارة الطلبات.
* تغيير حالات الطلبات.
* التحكم الكامل بالنظام.

---

# قاعدة عمل النظام

1. يقوم العميل بتسجيل الدخول.
2. يستعرض المنتجات المتاحة.
3. يختار المنتجات المطلوبة.
4. يتم إنشاء الطلب بحالة **قيد الانتظار**.
5. تقوم الإدارة بمراجعة الطلب والموافقة عليه.
6. يتم تحديث حالة الطلب إلى **تم الشحن**.
7. عند استلام العميل للمنتج يتم تغيير الحالة إلى **تم التسليم**.
8. يستطيع العميل متابعة حالة الطلب في أي وقت من خلال حسابه الشخصي.

---

# أهداف النظام

* تسهيل عمليات البيع والشراء إلكترونياً.
* توفير وسيلة آمنة لإدارة الطلبات.
* تمكين الإدارة من التحكم الكامل بالمستخدمين والمنتجات.
* متابعة حالة الطلبات بشكل مستمر.
* تحسين تجربة المستخدم وتوفير واجهة سهلة الاستخدام.
* توفير نظام صلاحيات يضمن حماية البيانات وإدارة النظام بكفاءة.
